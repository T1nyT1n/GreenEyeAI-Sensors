# Концепция

## Цель

**Создать часть физической инфраструктуры проекта для мониторинга состояния гидропоники** — автоматизированный модуль на основе ESP-8266, который должен:

- Считывать *данные с датчиков*
    - pH (уровень кислотности),
    - температуры,
    - света,
    - уровня влажности.
- Отправлять на *сервер*, который делает команда 1.

## Задачи

### Подготовка

#### Составить инструкцию для начала работы с ESP.

- Установка драйвера для ESP-8266.
- Установка редактора кода.
- Настройка среды разработки.

### Сборка

#### Подключение датчиков к ESP.

- Датчик pH
- Датчик температуры
- Датчик света
- Датчик уровня влажности

#### Поместить ESP и датчики в гидропонику.

- Найти оптимальное место для модуля.
- Обеспечить питание.
- Поместить модуль на место.
- Включить. 

### Сбор данных

#### Считывание данных с датчиков:

- pH (уровень кислотности),
- температуры,
- света,
- уровня влажности.

### Соединение с сервером

#### Подключение ESP к серверу.

- Использовать модуль Wi-Fi для связи с сервером на Raspberry Pi.
    - Настроить точку доступа Wi-Fi.
    - Использовать логин и пароль для подключения.

#### Отправка данных с помощью HTTP на сервер.

- Написать код для формирования HTTP-запросов из шаблонов.
- Обеспечить регулярную отправку запросов по Wi-Fi.
- Согласовать работу с сервером с 1-ой командой.
- Составить документацию по запросам между ESP и сервером, чтобы работать с 1-ой командой.
- Обспечить защиту от отключения — модуль может переподключиться к серверу в случае разрыва соединения.

## Окружение и инструментарий

### Аппаратные средства

- ESP-8266
- Датчик pH для Arduino
- Датчик температуры
- Датчик света
- Датчик уровня влажности
