# Документация прошивки ESP8266: взаимодействие с сервером

## Отправка данных на сервер

### Формат данных

Для каждого датчика формируется JSON-объект следующего вида:

```json
{
  "sensor_id": "0",
  "data": 123.45
}
```

* `sensor_id` – строковый идентификатор датчика. В текущей реализации используются значения `"0"`, `"1"`, `"2"` (соответствуют датчикам pH, освещённости и температуры).
* `data` – текущее показание датчика (число с плавающей точкой).

### HTTP POST запрос

Отправка производится функцией `sendSensorData()`. Полный HTTP‑запрос выглядит следующим образом:

```
POST /data HTTP/1.1
Host: <адрес_сервера>
Content-Type: application/json
Connection: keep-alive
Content-Length: <длина_тела>

<тело_JSON>
```

* **Адрес сервера** и **порт** задаются глобальными переменными `host` и `port` (по умолчанию `"10.21.36.131"` и `5000`).
* **Заголовок `Connection: keep-alive`** присутствует, но фактически соединение не переиспользуется – после отправки оно сразу закрывается.
* **Тело запроса** содержит JSON, описанный выше.
